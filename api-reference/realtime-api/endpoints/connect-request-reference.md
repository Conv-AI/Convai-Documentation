`ConnectRequest` is the JSON body sent to `POST /connect`. Only `character_id` is required. Every other field uses the default shown when omitted. Fields are grouped by function.

## Character and transport

| Field | Type | Required | Default | Constraints | Description |
|---|---|---|---|---|---|
| `character_id` | `string (UUID)` | Yes | — | Valid UUID v4 | The character to connect to. The character must be owned by the API key's account or have public or unlisted visibility. |
| `transport` | `string` | No | `"livekit"` | `"livekit"`, `"websocket"` | Realtime transport protocol. See [Choose a transport](../transports.md). |
| `connection_type` | `string` | No | `"audio"` | `"audio"` or `"video"` | Whether the session carries audio only or audio and video. |

{% hint style="info" %}
`transport` is not included in the autogenerated JSON schema for this model, but the field is accepted and validated at runtime. The default is `"livekit"`.
{% endhint %}

## Session control

| Field | Type | Required | Default | Constraints | Description |
|---|---|---|---|---|---|
| `mode` | `string` | No | `"create"` | `"create"` or `"join"` | Whether to create a new session room or join an existing one. Used with `shared_session_key` for multi-participant rooms. |
| `spawn_agent` | `boolean` | No | `true` | — | When `false`, the server provisions the room and returns credentials but does not start the AI agent. The caller is responsible for agent lifecycle. |
| `character_session_id` | `string \| UUID` | No | `null` | — | Pass a prior `character_session_id` to resume conversation history. The session must be owned by the same account. See [Resume a conversation](resume-conversation.md). |
| `room_name` | `string` | No | `null` | — | Request a specific LiveKit room name. When omitted, the server assigns one. |
| `shared_session_key` | `string` | No | `null` | 1–128 characters; alphanumeric, hyphen, and underscore only (`[a-zA-Z0-9_-]+`) | Client-supplied key that routes multiple participants into the same room. See [Shared sessions](shared-sessions.md). |
| `max_num_participants` | `integer` | No | `1` | — | Maximum number of participants the room accepts. Set above `1` when using `shared_session_key` for multi-participant sessions. |

## Language model provider

| Field | Type | Required | Default | Allowed values | Description |
|---|---|---|---|---|---|
| `llm_provider` | `string` | No | `"dynamic"` | `"gemini-live"`, `"gemini-live-beta"`, `"gemini-baml"`, `"dynamic"` | LLM provider for the session. `"dynamic"` lets Convai select the provider based on account configuration. |

## Blendshape and facial animation

| Field | Type | Required | Default | Allowed values | Description |
|---|---|---|---|---|---|
| `blendshape_provider` | `string` | No | `"not_provided"` | `"not_provided"`, `"ovr"`, `"neurosync"` | Facial blendshape provider. Use `"neurosync"` to receive NeuroSync blendshape frames during the session. |
| `blendshape_config` | `object` | No | `null` | See below | Provider-specific blendshape configuration. Required when `blendshape_provider` is `"neurosync"`. |

**`NeurosyncConfig` fields** (used when `blendshape_provider` is `"neurosync"`):

| Field | Type | Default | Description |
|---|---|---|---|
| `provider` | `string` | `"neurosync"` | Must be `"neurosync"`. |
| `enable_chunking` | `boolean` | `true` | Send blendshape frames in chunks. |
| `chunk_size` | `integer` | `10` | Number of frames per chunk. |
| `output_fps` | `integer` | `60` | Client delivery FPS until NeuroSync ACK reports the server FPS. |
| `frames_buffer_duration` | `float` | `0` | Buffer duration in seconds. |
| `format` | `string` | `"MHA"` | Blendshape frame format. |

## End user

| Field | Type | Required | Default | Constraints | Description |
|---|---|---|---|---|---|
| `end_user_id` | `string` | No | `null` | Whitespace is stripped | Stable identifier for the end user of this session. Required when the character has Long Term Memory enabled. |
| `end_user_metadata` | `object` | No | `null` | `name` key must be a `string` or `null` | Arbitrary metadata attached to the end user. Ignored if `end_user_id` is not provided. The `name` key is persisted to the speaker record and drives transcript prefixes. |

{% hint style="warning" %}
When `end_user_id` is omitted and `end_user_metadata` is provided, the server silently ignores `end_user_metadata`.
{% endhint %}

## Dynamic context and narrative

| Field | Type | Required | Default | Description |
|---|---|---|---|---|
| `dynamic_info` | `object` | No | `null` | Per-session context text injected into the character's prompt. Contains `text` (string), `static_text` (string), and `keep_in_context` (boolean, default `false`). Subject to token budget limits. |
| `narrative_template_keys` | `object` | No | `{}` | Key-value pairs that fill named slots in the character's narrative design templates. |

## Actions and scene

| Field | Type | Required | Default | Description |
|---|---|---|---|---|
| `action_config` | `object` | No | `null` | Configures actions the character can perform, including `actions`, `characters`, `objects`, and `current_attention_object`. |
| `scene_description` | `array of objects` | No | `null` | List of objects in the scene. Each item has `name` (string) and `description` (string). |

## Audio

| Field | Type | Required | Default | Description |
|---|---|---|---|---|
| `audio_config` | `object` | No | `null` | Audio output configuration. Only supported with LiveKit transport. See [Audio output configuration](audio-config.md). |
| `vad_params` | `object` | No | See [VAD parameters](vad-params.md) | Voice activity detection parameters for this session. See [VAD parameters](vad-params.md). |
| `default_tts_enabled` | `boolean` | No | `true` | Whether text-to-speech output is enabled at session start. |
| `default_stt_enabled` | `boolean` | No | `true` | Whether speech-to-text input is enabled at session start. |

## Thinking and turn detection

| Field | Type | Required | Default | Description |
|---|---|---|---|---|
| `thinking_mode` | `boolean` | No | `false` | Enable extended reasoning mode for the LLM. Increases response latency. |
| `turn_detection_config` | `object` | No | `null` | Smart turn detection configuration. Contains `type` (`"smart_turn"` or `"not_provided"`) and optional `params`. |

## Video

| Field | Type | Required | Default | Description |
|---|---|---|---|---|
| `video_track_name` | `string` | No | `"camera"` | Name of the video track published to the room when `connection_type` is `"video"`. |

## Speech-to-text

| Field | Type | Required | Default | Description |
|---|---|---|---|---|
| `stt_provider` | `string` | No | `null` | Override the default STT provider for this session. When `null`, Convai selects the provider based on account configuration. |
| `disable_live_fallback` | `boolean` | No | `true` | Disable fallback to an alternative LLM provider when the primary is unavailable. |

## Emotion

| Field | Type | Required | Default | Description |
|---|---|---|---|---|
| `emotion_config` | `object` | No | `null` | Emotion detection configuration. Accepts `{ "provider": "nrclex" }` or `{ "provider": "llm" }`. |
